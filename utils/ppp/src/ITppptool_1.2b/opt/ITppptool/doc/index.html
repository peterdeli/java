ITppptool for Solaris

Introduction


PPPtool is a graphical interface for connecting to remote networks via dialup modem.

The Main window consists of a Menu, a display of active account information,
the 'Connect/Disconnect' Button, some mock 'status lights', and a 'status' field.

Upon first launching PPPtool, the following window will appear :


 

The Menu bar contains four menus:

1. 'File'
2. 'Edit',
3. 'Accounts'
4. 'Help'.

The 'File' menu contains three items:

1. Save Accounts
2. View Log
3. Quit

The 'Edit' menu contains two items:

1. Manage Accounts
2. Edit Preferences


 

The 'Accounts' menu is a dynamic menu and holds the names of all
 accounts which have been created:


 

The 'Help' menu contains two items:
1. 'About PPP Tool'

ITppptool Configuration
 
 

I. Creating Accounts

To create a new account, Select 'Edit/Manage Accounts, then click on 'New' ( see below ):


 


 
 

A small window will appear with a pop-down menu with two choices

ISP_ModemPool
Sun_ModemPool

The ISP_ModemPool selection is for all accounts which do not connect directly to Sun.
The Sun_ModemPool selection is for a Sun modempool account which connects directly to Sun.
 

Make your choice and click the 'Continue->' button


 

A window will appear with the following fields: ( ** denotes a required field )

[ ** Account Name ]: Enter the name of the account
[ ** User ID ]: Enter the login name or ID of the account
[ Password ]: Enter the password of the account ( if one exists )
[ ** Phone Number ]: Enter the Phone Number of the account
[ ** DNS Domain Name ]: Enter the domain name of the account ( if one exists )
[ DNS Search Domains ]: Enter any domains which DNS should search
[ ** Account Nameserver #1 ]: Enter a primary nameserver for the account
[ Account Nameserver #2 ]: Enter a secondary nameserver for the account

Near the bottom of the window are the following checkbuttons:

[ Default Route ]: If selected, adds a 'default route' option to pppd
[ ** Create /etc/resolv.conf ]: Click on one of the buttons:
    [ Do not create ] - this will not create an /etc/resolv.conf file, or overwrite an existing one.
    [ Use peer DNS ] - this will add any nameservers returned by the remote server to the /etc/resolv.conf file.
    [ Use account DNS ] - this will use the information in the account for the /etc/resolv.conf file.

The entry box 'PPP options' is for special configuration of the ppp connection.
See the pppd man pages for details. Examples options are novj, nopcomp.

The 'Debugging' checkbox will enable more verbose output to the log window
during the startup of the ppp connection.

II. Saving Accounts

To save the current account information to the account file, select File/Save Accounts.
If there have been any changes to the accounts, a prompt will pop up to confirm the save.


 

Clicking 'Yes' will save the account data,
clicking 'Cancel' will not save the account data and return to PPP tool.

III. Editing Accounts

To make changes to an account, first select Edit/Manage Accounts
Click on an account name in the window, then click on 'Edit'
A window will appear with the account information:

Edit any fields or select/deselect any items, then click on 'Save Edits'
The window will close and the changes will be saved in the currently running program,
but will NOT be saved to the accounts file. To save the changes to the accounts file, select
'File/Save Accounts' and click on the 'OK' button if the 'Save' dialog pops up.
If the Save dialog does not pop up, there are no changes to save.

By not saving changes to disk when invoking 'Save Edits',  it is possible
to recover from inadvertent edits to your account.
To recover from an error during an edit, select File/Quit, and click
'No' when asked to save changes. Then restart PPPtool and the original
configuration will be restored.


 

IV. Deleting Accounts

To delete an account, first select Edit/Manage Accounts.
Click on an account name in the window, then click on 'Delete'.

After clicking the 'Delete' button, a prompt will appear asking
to confirm the request. Click 'Yes' to delete the account,
or 'No' to cancel.

The account will be deleted in the currently running program, but will NOT
be saved to the accounts file on the local disk.
To save the changes permantently, select 'File/Save Accounts' and
click on the 'OK' button of the window which appears.


 
 

If you have accidentally deleted an account, immediately select 'File/Quit',
and click 'No' when asked to save changes. This will cause the changes not
to be saved permanently. Restarting PPPtool will restore the deleted account.
 


 

V. Making an  Account Active

To make an account active makes it available for connecting.
There are three ways to do this:

1. Select the account from the 'Accounts' menu item in the main window.

2. In the 'Account Manager' window, select the account and click the 'Make Active' button.
3. In the 'Account Manager' window,  double-click on the account.

VI. Preferences

Upon first starting ppptool, a set of default preferences will be loaded.
To make sure the correct settings are in effect, check the preferences window by selecting 'Edit/Edit Preferences'.

The settings in the preferences are:

Modem Port: (required): This is the port to which the modem is connected.
Flow Control: hardware, software, or none.
Port Speed (required)
Init String: Enter any modem initialization string here.
 


 

Connecting with PPPtool


To initiate a  ppp connection, follow these steps:

1. Select an account and make it active ( See V. above )
2. Make sure your modem is connected to the correct port,
    and that the correct cable is connected between the modem and the computer.
3. Verify the account information is correct, then click on the 'Connect' Button.

4. If possible, turn up the volume on the modem so that you can hear it
    going off-hook, dialing, and connecting.
5. Monitor the 'Status' line at the bottom of the main window, and the
    log window as well ( File/View Log ).

The order of execution to connect is:

1) Initialize modem
2) Dial phone number and connect
3) Start the ppp service


 
 

 Connecting to Sun Modempools  with a tokencard


For Sun Modempool connections, there will be instructions during the
connection on when to key in numbers into your tokencard, as well as
when to enter to tokencard responses into the ppptool tokencard dialog.

First, click the 'Connect' button. PPPtool will initialize the modem and
then dial the modempool number. During this time, a window will appear
with some instructions to prepare your tokencard by entering the tokencard
password, and waiting for a 'challenge' ( consisting of '9' followed by four numbers. )
 


 

If the initial authentication is successful, PPPtool will be connected
and will then receive a 'Challenge' sequence of numbers from the
ModemPool Server. This will be displayed as in the 'TokenCard Challenge/Response' window
below. Enter these numbers into the tokencard and press 'E' as in the
instructions in the window. The tokencard will then respond with a string
of alpha-numeric characters. Enter the string into the TokenCard Challenge/Response field and
click 'Send Response' or press 'Enter'.


 

If the response entered into the TokenCard Challenge/Response window is correct,
PPPtool will be logged into Sun:


 
 

If no response is entered within the time limit, the PPPtool log window will indicate this:


 
 

To end the connection, click on the 'Disconnect' button. PPPtool will then terminate the connection.

Exiting PPPtool

To exit PPPtool, select 'File/Quit'.
 

If there is any account information which has not been save to the account file,
a dialog will pop up asking whether to save and exit, exit without saving,
or cancel the exit and  return to PPPtool.

Clicking 'Yes' will save the account settings to the file.

Clicking 'No' will exit PPPtool without saving the account changes, and
clicking 'Cancel' will return to PPPtool without saving any changes to the
accounts file.
